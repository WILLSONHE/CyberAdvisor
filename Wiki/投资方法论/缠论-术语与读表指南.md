# 缠论 · 术语与读表指南

> **用途**：读懂 sug / 看板 / 模拟盘 / 回测里出现的缠论字段。  
> **重要**：本项目 `scripts/chan/` 是 **108 课思想的启发式自动化**，输出带 **「候选」** 字样 ≠ 经典定义下已确认的买卖点。  
> 方法论原文：[[缠论]] · [[缠论-背驰与买卖点]] · 实现细节：[[缠论-数据接入]]

---

## 一、一行快照怎么读？

看板总览、标的池日报、模拟盘 tick 里常见 **一行摘要**：

```text
缠论 上涨趋势 | 观望/空 | 动作=wait | 保护≈4042.18 | 分=-1.2
```

| 片段 | 字段名 | 含义 |
|------|--------|------|
| `上涨趋势` / `盘整` / `下跌趋势` | **structure** / **trend_day** | 近 60 根 K 线高低点关系判定的 **走势类型标签**（见 §二） |
| `一买候选` / `观望/空` / … | **buy_point** | **买卖点提示**（见 §三），与走势类型 **独立计算** |
| `动作=wait` | **action** | 系统建议的操作码（见 §四） |
| `保护≈4042` | **protect_price** | 结构失效参考价（见 §五） |
| `分=-1.2` | **score** | 门禁与排序用的数值（见 §六） |

**这些字段回答不同问题，可以同时出现看似「矛盾」的组合**——见 §七。

**操作指引**：报告与看板在结论旁自动附带 **`操作指引`** 一句（代码 `scripts/chan/guidance.py`），按 `buy_point` + 是否持仓 + 走势类型生成，例如：

> 近段走势仍标注为**上涨**（高低点整体抬高），但现价已落至中枢下方，处于**观望/空**阶段（收盘低于 ZD≈…）。**已有持仓**：**不应开新仓/加仓**；保护位 **xxx.xx**——有效跌破则多头结构失效，宜**减仓或至少停止加仓**；…

详见各 `buy_point` 在 §三、§七 与代码模板。

## 二、走势与结构

| 术语 | 在本项目里指什么 | 经典缠论对照 |
|------|------------------|--------------|
| **上涨** | 近段 **低点抬高 + 高点抬高**（或收盘在中枢上方） | 上涨走势类型 |
| **下跌** | 低点降低 + 高点降低 | 下跌走势类型 |
| **盘整** | 高低点无明确同向序列，或收盘贴近区间中部 | 盘整走势 / 中枢震荡 |
| **structure** | 展示用：`盘整` 或 `上涨趋势` / `下跌趋势` | 走势类型文字版 |
| **ZD** | 近 40 根 K **低点分位 25%** 的近似 **中枢下沿** | 中枢下沿 ZD（简化） |
| **ZG** | 近 40 根 K **高点分位 75%** 的近似 **中枢上沿** | 中枢上沿 ZG（简化） |
| **中枢** | 本脚本 **未** 严格递归划分笔/线段；用 ZD–ZG 区间 **代替** | 需人工画结构时仍以 [[缠论]] 为准 |

> ZD/ZG 是 **统计近似**，不是逐笔划分的中枢。除权后请用 **前复权** K 线（见 [[缠论-数据接入]] §1.2）。

## 二·五、走势运动用语（回调 / 回抽 / 反弹…）

报告与 `buy_reason` 里常出现下列词，**不是独立字段**，而是对 K 线运动的描述：

| 术语 | 含义 | 在本项目里何时出现 |
|------|------|-------------------|
| **回调** | 在**已有上升或反弹**之后，价格**向相反方向**回落一段（一般不破关键结构） | **二买候选**：上涨/盘整中回落，但未破前低 |
| **回抽** | 价格**离开某一关键位**（常为 ZG/中枢上沿）后，**再次回到该位附近**测试支撑，**不破**则视为确认 | **三买候选**：站上 ZG 后回落，低点仍 ≥ ZG |
| **反弹** | 在**下跌**之后出现的**上涨**波段（未必反转趋势） | 一买候选语境：下跌末端的向上修复 |
| **探底** | 继续下探寻找更低支撑 | 下跌段、观望/空阶段 |
| **创新高 / 创新低** | 相对对比窗口内的高/低点被刷新 | MACD **背驰**比较的两段价格 |
| **离开中枢** | 收盘价**有效站上 ZG**（本项目简化为 > ZG×1.005） | 三买候选前置 |
| **中枢震荡** | 价格在 ZD~ZG 区间内来回 | **无明确买卖点**、部分 **盘整** 标签 |
| **顶背驰 / 底背驰** | 价创新高新低，但 MACD 柱**面积**未同步放大 | 一卖/一买候选的 MACD 依据 |

**回调 vs 回抽（口诀）**：

- **回调**：大方向仍多（或仍空）时的**顺势回撤** → 二买里「涨完歇一歇」。  
- **回抽**：**突破关键位之后**的「回头确认」→ 三买里「离开 ZG 后再踩一脚 ZG」。

## 三、买卖点标签（buy_point）

| 标签 | 触发条件（代码逻辑） | 操作含义 |
|------|----------------------|----------|
| **一买候选** | 日线标为 **下跌** + MACD 柱面积 **底背驰**（价创新低、面积未创新低） | 趋势末端 **可能** 反转；需上级/区间套确认 |
| **二买候选** | 上涨/盘整中，回调 **未破前低** 且价在中枢上沿下方；可选 60min 背驰 | 趋势延续中的 **回调买点** |
| **三买候选** | 价 **站上 ZG** 后回抽，低点 **未跌破 ZG** | 离开中枢后的 **回抽确认** |
| **一卖/减仓** | 日线 **上涨** + MACD **顶背驰** | 持筹者考虑减仓；指数出现时常 **禁止新开仓** |
| **观望/空** | 收盘价 **低于 ZD**（落在中枢下方） | 结构偏弱；**空**=不做多/偏防守，**不是**系统自动开空单 |
| **无明确买卖点** | 以上皆不满足 | 中枢内震荡或结构未完成 |

### 为什么叫「候选」？

经典缠论的一/二/三买需：**包含处理 → 笔 → 线段 → 中枢 → 背驰 → 级别联立**。  
自动化脚本只做 **日 K + 可选 60min + MACD 面积** 的 **启发式筛选**，故一律加 **「候选」**，最终是否建仓仍受：

- **指数缠论门禁**（上证一卖等）
- **七轨位置**（`can_open`）
- **Wiki 指数纪律**（4033/4120 等）

## 四、建议动作（action）

由 `buy_point` + **是否已有持仓** 映射而来：

| action | 无持仓时 | 有持仓时 |
|--------|----------|----------|
| **buy** | 可考虑新开仓（仍须过门禁） | — |
| **hold_add** | — | 持有，可小幅加仓 |
| **hold** | 观望 | 持有 |
| **wait** | 不介入 | 不主动加仓 |
| **sell** | — | 减仓/清仓倾向 |

> **「观望/空」+ action=wait**：没有仓 → 不开新多；有仓 → 映射为 **sell**（倾向减仓）。

## 五、保护位（protect_price）

**保护位** = 若价格 **有效跌破** 该价位，则当前 **买点叙事/结构假设失效**，应停止按原逻辑持有或加仓。

| 买点类型 | 保护位取法 |
|----------|------------|
| 默认 / 一买 | 约 **ZD**（中枢下沿） |
| 二买 | 优先取 **背驰段前低**（`divergence_down.price2`） |
| 三买 | 约 **ZG**（回抽不能破的上沿） |

**模拟盘 / 门禁**：现价低于 `protect_price × 0.998` 可触发 **强制卖出** 判断（`should_force_sell`）。  
**outlook 融合**：1/3/7 日区间下沿也会参考保护位 **抬高 floor**（`outlook_blend`）。

**「保护」不是止盈价**，而是 **结构止损参考**；与 Wiki 里的「4033 清仓线」等 **指数纪律** 是不同层。

## 六、评分（score）

| buy_point | 基础分 |
|-----------|--------|
| 三买候选 | +2.0 |
| 二买候选 | +1.6 |
| 一买候选 | +1.0 |
| 无明确买卖点 | 0 |
| 观望/空 | -1.2 |
| 一卖/减仓 | -1.8 |

- 模拟盘买入排序：`缠论 score × 2.5` **优先于** 布林分。  
- `allows_new_buy`：无买点且 score ≤ 0 → **不允许新开仓**。

## 七、常见「矛盾」组合（FAQ）

### Q1：`观望/空` 和 `上涨` 为什么同时出现？

**因为它们来自两条独立规则：**

| 维度 | 规则 |
|------|------|
| **上涨**（trend_day） | 看 **最近 60 根** 高低点序列：可能整体仍「低点抬高」 |
| **观望/空**（buy_point） | 看 **当前价 vs ZD**：只要 **收盘 < ZD** 就标空，**不管** trend 标签 |

**典型场景**：中期反弹后 **最后几根 K 线回落**，趋势标签仍显示「上涨」，但价已 **跌回中枢下方** → 系统提示 **观望/空**，表示 **不宜按多头结构继续加仓**。

### Q2：`一买候选` 但 action 是 `wait`？

可能原因：

1. **无持仓** 且 **指数一卖** / `allows_new_buy=false`  
2. **七轨** 过高，`can_open=false`  
3. 仅有「候选」，门禁未过——**候选 ≠ 可建仓**

### Q3：`保护≈4042` 和现价差很多？

保护位跟 **买点类型** 走，不是现价。指数在 **4033 纪律线** 附近时，ZD 算出的保护位可能与 Wiki **4033/4120** 并存——**指数纪律优先于 outlook**，缠论 **hard_gate** 优先于 LLM 建议。

### Q4：「空」会开空单吗？

**不会。** A 股默认 **只做多**；「空」= **不做多 / 偏空观望**。模拟盘与 sug 均 **不下融券空单**。

### Q5：底背驰 / 顶背驰 是什么？

比较 **两段 MACD 柱面积**（`chan/indicators.py`）：

- **底背驰**：价 **创新低**，面积 **未创新低** → 下跌动能衰竭（一买区）  
- **顶背驰**：价 **创新高**，面积 **未创新高** → 上涨动能衰竭（一卖区）

## 八、门禁与报告用语

| 术语 | 含义 |
|------|------|
| **allows_new_buy** | 缠论 **第一门禁**：标的 + 指数结构是否允许 **新开仓** |
| **can_open** | sug 研判总结：综合七轨 + 缠论 + 指数后的 **可否建仓** |
| **open_block_reason** | 被挡原因，如 `指数缠论：一卖/减仓` |
| **hard_gate** | Graph 管线最后一步；**PM/LLM 不能绕过** |
| **指数一卖** | 上证等 `buy_point` 含 **一卖/减仓** → 全员 **禁止新开多**（已有仓另论） |

## 九、回测专用术语

看板 **「缠论回测」** 与 `Wiki/数据/缠论回测/` 报告使用：

| 术语 | 含义 |
|------|------|
| **回测** | 用 **历史日 K**，在每个交易日 T 只用 **T 及以前** 的数据跑 `analyze_day_frame`，**禁止偷看未来** |
| **walk-forward** | 按时间逐日向前滑动；不是「用全历史算一次信号」 |
| **独立信号** | 同一标的、同一日、同一 `buy_point` 计 **1 次**（多持有周期共享） |
| **命中** | 信号后 N 日内：最高价涨幅 ≥ **阈值（默认 3%）** 且 **最低价未破保护位** |
| **破保护率** | N 日内 **最低价 < protect_price** 的比例 |
| **命中率** | 命中次数 / 信号次数 |
| **IS / OOS** | 样本内 / 样本外：按信号日期 **后 30%** 划为 OOS，检验是否过拟合 |
| **一买/二买 分行统计** | 不同 `buy_point` 标签 **分开** 计命中率（启发式规则质量不同） |

> 回测 **仅验证自动化规则** 的历史表现，**不等于** 实盘收益承诺；二买候选命中率通常 **高于** 一买候选（见最新 `.md` 报告）。

## 十、与经典缠论差异（必读）

| 项目 | 经典缠论 | 本项目自动化 |
|------|----------|--------------|
| 中枢 | 笔/线段递归 | ZD/ZG 分位数近似 |
| 买卖点 | 严格定义 + 级别联立 | 规则筛选 + **「候选」** |
| 包含/笔 | 人工或完整算法 | **未实现** 完整包含处理 |
| 操作级别 | 用户选定 | 默认 **日线 + 60min** 联立 |

**结论**：报告里的缠论块 = **统一门禁语言** + **结构提示**；精细画结构、定级别仍以 [[缠论-实战法]] 与人工为准。
