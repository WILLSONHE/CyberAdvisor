# 缠论 · 数据接入与优先级

> **系统约定**：所有 `sug` / `qry` / 模拟盘 tick / `analysis_report.py` 输出中，**缠论结构优先于**布林七轨、outlook 概率、RS。  
> 代码入口：`scripts/chan/` · 报告嵌入：`report_data.enrich_stock` · 模拟盘：`ai_sim/collector` + `strategy.py`

---

## 1. 缠论最低数据需求

| 层级 | 周期 | 用途 | 最少根数 |
|------|------|------|----------|
| 上级 | 日线 | 趋势/中枢/一买 | ≥120 |
| 操作级 | **60min**（或 30min） | 二买/三买/区间套 | ≥120 |
| 精确定位 | 5min / 1min | 区间套、盘中确认 | 见 §1.1 |

### 1.1 本地 1/5 分钟要下多长？

A 股约 **240 根/日（1min）**、**48 根/日（5min）**。

| 周期 | 最低（能跑） | **推荐（区间套+复盘）** | 对应场景 |
|------|--------------|-------------------------|----------|
| **5min** | **10 交易日**（≈480 根） | **60 交易日**（≈3 个月，2880 根） | 覆盖多个 60min 中枢段 |
| **1min** | **5 交易日**（≈1200 根） | **20 交易日**（≈1 个月，4800 根） | 近端区间套精确入场 |

> **mootdx 在线** 5min 单次最多约 **800 根（≈17 交易日）**——实盘够用，深度回测仍靠通达信本地补历史。  
> 通达信补数据：**选项 → 系统设置 → 设置1 → 维护/下载** → 勾选 **1分钟线、5分钟线** → 对 **000001 + 持仓 + 追踪池** 执行。

## 1.2 复权：前复权 vs 不复权

| 类型 | 含义 | 缠论影响 |
|------|------|----------|
| **不复权** | 真实成交价；除权除息日 **跳空缺口** | 易出 **假分型/假笔** |
| **前复权**（推荐） | 历史价向前调整，K 线 **连续** | 结构划分稳定，与 MACD 一致 |
| 后复权 | 现价向前调整 | 本 Wiki **不用** |

**在哪里选（通达信 PC，`C:\new_tdx64`）**：

1. **全局默认**：`选项` → `系统设置` → `设置3`（或「外观/图表」）→ **缺省复权类型** → 选 **前复权**  
2. **单图切换**：K 线图 **右键** → `复权` → `前复权`  
3. **下载到 vipdoc 的 .day / minline** 随客户端当前复权设置写入——改复权后建议 **重新补下载** 日 K 与分钟线。

**mootdx 在线 K** 与 vipdoc **须同一复权口径**；混用会在除权日前后结构不一致。本项目 **推荐统一前复权**。

## 2. 自动采集（现有端口）

| 数据 | 端口/模块 | 覆盖 | 说明 |
|------|-----------|------|------|
| 日 K | `daily_bars.py` → vipdoc → **Tushare qfq** → mootdx | 持仓+追踪池+指数 | `.env` 设 `TUSHARE_TOKEN`（2000 积分）；**仅补不足/缺最新 bar**，历史仍以 vipdoc 为主 |
| 日 K（本地） | `tdx_vipdoc` / `tdx_market_data.read_daily_bars` | 同上 | `TDX_VIPDOC` 本地 `.day` + `ds/lday` |
| 60/30/5/1min | `chan/kline.py` → **mootdx** `Quotes.bars` | 全 A 股（在线） | category 3/2/0/7 |
| 1/5min 本地 | `read_minute_bars` | **仅通达信已下载**的票 | 通常近 10~50 交易日 |
| MACD+背驰 | `chan/indicators.py` | 自动 | 柱面积比较 |
| 结构快照 | `chan/analyze.py` | 日+60min 联立 | 一/二/三买候选、ZD/ZG、action/score |
| **outlook 1/3/7** | `chan/outlook_blend.py` → `bollinger_utils` | 区间/概率中心/挡位 | 融合保护位、ZD/ZG；参数 `Wiki/数据/股价预测追踪/参数.json` |
| 报告 | `report_data` / `bollinger_utils.build_stock_verdict` | sug/qry | 每标的缠论块在布林**之前** |
| 模拟盘 tick | `ai_sim/collector` | 标的池每只 | JSON 字段 `chan` |
| 模拟盘决策 | `ai_sim/strategy` | 买入排序 | 缠论 score×**2.5** > 布林×0.25 |

**CLI 自检**：

```bash
python -c "import sys; sys.path.insert(0,'scripts'); from chan.analyze import analyze_code; print(analyze_code('600021'))"
python scripts/tushare_daily.py --code 600021 --json   # 日 K 三源自检
```

## 3. 需你手动接入/维护

| 项 | 操作 | 原因 |
|----|------|------|
| **通达信 vipdoc** | PC 客户端每日收盘后同步；`.env` 设 `TDX_VIPDOC` | 本地日 K / 短历史 1·5 分依赖客户端写入 |
| **Tushare 日线** | `.env` 设 `TUSHARE_TOKEN`；`python scripts/tushare_daily.py --code 600021` | vipdoc 未同步或新票时 **自动补前复权日 K**（2000 积分 `pro_bar qfq`；**不含分钟**） |
| **1/5 分钟长历史** | 通达信 → 系统 → 数据维护 → 补充 **1min/5min**（持仓+指数） | 本地 lc1 默认仅 ~10 日；区间套回测不够 |
| **复权一致** | 通达信统一 **前复权**（见 §1.2） | 除权假分型 |
| **mootdx 服务器** | 已由 `daily.bat` → `mootdx_bestip.py` 每日测速 | 见 §1.3 |
| **东方财富分钟** | 可选；当前 **不稳定** | `supplement.fetch_stock_kline_em` 勿作唯一源 |
| **涨跌停/停牌** | 暂无自动字段 | 一字板笔/线段失真，需人工过滤 |
| **期货** | `TDX_VIPDOC_QH` 未接入 | 脚本仅 A 股 |

## 5. 数据流

```
通达信 vipdoc (.day / minline)     ← 主（前复权，与 TDX 一致）
        ↓ 根数不足 / 缺最新交易日
Tushare pro_bar qfq（TUSHARE_TOKEN）
        ↓ 仍不足
mootdx 在线 (30/60/5/1min；日 K category=4)
        ↓
chan/kline → indicators → analyze
        ↓
┌───────────────┬────────────────┐
│ sug / qry     │ ai_sim tick    │
│ report_data   │ strategy 排序  │
└───────────────┴────────────────┘
```
