# sug 交易策略回复模板

每次用户说 `sug {持有人}`、`sug {持有人} 早盘|午盘`、`sug 全员`，或询问某持有人的交易策略/持仓/开仓时，严格按此模板输出。

**表述规范**：正文须用 Wiki / 研报 / 日更 / 标的池日报 等可核查出处，并标明来源（见 `SKILL.md` → **表述规范**）。

**盘次语义**：`早盘` = 上午收盘后的 **回顾**；`午盘` = 全天收盘后的 **回顾**（含实际收盘点位与 4033/4130 是否触发）。盘次总结 **禁止** 使用「收盘前必盯」「午盘至收盘必盯」等前瞻表述。

归档：`SugVault/YYYY-MM-DD_{持有人}_sug.md`；若指定早盘/午盘则为 `..._sug 早盘.md` / `..._sug 午盘.md`。
持仓数据以 `portfolio.md` 对应持有人章节为准（`持仓.xlsx` → 同步）：**投资成本 = 成本 × 股数**，**市值 = 现价 × 股数**（含交易成本计入成本侧）。二、三、五章金额仅统计该持有人；第二章须分别列出投资成本与市值两列总计。

**金额格式**：凡 **元** 为单位的投资成本、市值、现金、成交额、仓位金额等，须 **千位逗号分隔 + 两位小数**，例 `90,000,000.00`、`16,353.00`（单价/指数点位除外）。

## 七、研判总结

> **必写章节**（sug / 单标的分析 / 含持仓的诊断报告均须包含）。解释上文数据：**现价相对 20MA 与七轨位置（高/低/中枢）**、**宜/忌操作**、**能否建仓**、**持仓处置与权重参考**、**1 日 / 3 日 / 7 日技术倾向**（非盈利预测）。若数据不足，在「数据缺口」列出并指向 `scripts/ai_sim/supplement_registry.yaml` / Agent `data_requests`。

**生成方式**（Agent 或本地）：

```bash
python scripts/analysis_report.py --holder Wilson
python scripts/analysis_report.py --code 600021 --name 上海电力
```

也可在 Python 中调用 `scripts/bollinger_utils.py` → `build_report_summary_section(holdings=..., candidates=..., sh_index=...)`，将输出 **原样粘贴** 到本章（可微调措辞，但不得删字段）。

**本章须覆盖**（对每只持仓 + 主要候选各一段）：

- 七轨位置（顶/二/中/四/五/底）与相对 20MA 高低
- 适合做什么 / 不适合做什么（含做 T、追高等）
- 指数纪律下能否新开仓
- 有仓：清仓 / 持有 / 做 T / 权重 % 参考
- **1 日 / 3 日 / 7 日技术倾向**：须写 **结论 + 最有可能价位 + 推导过程 + 数据依据**（通道/MA 斜率/近10日高低/量能/**vipdoc σ**/收盘序列/支撑压力），并给参考区间；**区间上下沿须标注较现价 ±%**；**区间内各价格挡位**须列 **倾向概率表**（含 **概率加权漂移**；合计 100%，注明非真实概率）
- 数据缺口清单（分钟 K、RS、北向等）

**单标的分析报告额外要求（第三章 基本面）**：

- 须含 **直接竞争对手** 对比表（2–4 只，同细分或研报对标）
- 须含 **行业 benchmark**（同业中位数或龙头均值；注明样本与数据源）
- 对比维度至少：**PE/PB、ROE、毛利率、净利率、营收规模、近5/20日涨跌、布林位置**

- **专业名词解释（单标的 / 深度分析报告 · 必写）**：

- 须设独立章节 **「专业名词解释」**（或等价标题「术语与对本标的的含义」）
- 生成时调用 `scripts/report_glossary.py` → `format_glossary_markdown(b, outlook=…)`，或手工等价表格
- 对报告中出现的 **行业缩写、技术指标、纪律用语、vipdoc、最有可能价位** 逐条说明：**是什么 → 对本标的意味着什么（interpretation）**
- 至少覆盖：行业类、技术类（20日MA、带宽、七轨、锚点、最有可能价位、N字、做T）、财务类（ROE/PE/PB）、纪律类（4033/L1）、数据类（vipdoc、倾向概率）

**补充数据与数据缺口（必守）**：

- 生成报告前须调用 `scripts/report_data.py` → `enrich_stock(code)`，将 **财务 / vipdoc 本地日 K / 5分钟K / RS vs 沪深300 / 北向** 写入报告 **「补充数据」** 节
- **禁止**硬编码「分钟K / RS / 北向待补全」——仅当 `enrich_stock` 返回 `gaps` 非空时才写 **数据缺口** 节
- `gaps` 为空时写 **「数据覆盖：无未登记缺口」**（见 `format_gaps_markdown`）
- 行业叙事类（如医保政策进度）写在正文脉络节，**不列入数据缺口**

## 判断规则

### 回复 "不能"

- Wiki/日更口径为「减仓」「不追高」「等冰点」且冰点尚未出现
- 标的池中所有非科创板标的都在布林线一轨以上（已拉升过热）
- Wiki 框架判断大盘处于退潮期或明确看空

### 回复 "建议开"

- Wiki 口径为「分歧是加仓位置」「冰点后低吸」且冰点已现
- 有非科创板标的触发收敛信号/N 字二轨信号且 PE 在标的池中处于低位

### 回复 "能"

- Wiki 口径为「持股为主」「可以开仓」「上升趋势完好」
- 有非科创板标的在布林线二轨附近且属于标的池当前优先级前 3 的方向

### 其他情况 → "不建议"

### 科创板标的

- 如果唯一符合条件的标的是科创板的，诚实告知「唯一信号在科创板，你买不了，不开」
- 脚本输出的科创板标的在推荐表中自动过滤，但可在备注中提及

### ETF

- Wiki/日更明确提过的 ETF 纳入候选
- Wiki 未提过的不自己编
